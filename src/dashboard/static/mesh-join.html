Connect this device

This page only holds your coordinator and join code. It does not install Tailscale or finish mesh join by itself. Do the steps in order.

Before you start
  1. On a device already signed into your Cove (not this new phone/Mac), open Settings → Devices and tap Get join code (or open this page from that QR). You need that signed-in session to mint the code.
  2. Keep this page (or the join code) handy on the new device after install.
Step 1 — Install Tailscale on this device

Free Tailscale app. This is a separate private network for your Cove — not “log into Lucid Cove” in the browser alone, and not your Apple / Google / Microsoft account as the mesh login.

App Store (iPhone / iPad) Play Store (Android) Mac download

Already on another Tailscale network? Log out of that one first — Cove uses a separate network.

Step 2 — Alternate / custom server before any login

If you tap plain Log in first, Tailscale opens the default public network (Apple ID, Google, Microsoft, “IPN Extension”, etc.). Your Mac or iCloud password will not work for the Cove mesh. Stop and set the server first.

Do not sign in with Apple, Google, Microsoft, or your computer password. Those are for Tailscale’s public product. Cove uses the coordinator below plus a one-time join code.

iPhone / iPad: Open Tailscale → account / ⋯ menu → Use a custom coordination server or Use an alternate server → paste:

__LOGIN__

Mac: Open Tailscale → menu bar icon → plain Log in… is wrong if it jumps straight to Apple/Google. Prefer: Add account → small ▾ / arrow next to the button → Add Account Using Alternate Server (wording varies) → enter the same coordinator (with or without https:// as the field allows) → continue. Only after the alternate server is set should a sign-in / auth-key step appear.

Android: Same idea — custom / alternate coordination server before default log in.

Step 4 — Open your Cove as you
__COVE_BLOCK__
  1. Tailscale must show Connected first — the Cove address is private mesh-only.
  2. Then open the Cove (button above when present, or the sign-in link from Settings → Devices on a device that is already in).
  3. Sign in as you. Your personal Mission Control needs the mesh even when the public Cove door does not.
Step 5 — Keep this phone on the mesh (iPhone)

iPhone often suspends Tailscale in the background. When that happens every mesh Mission Control fails at once, then comes back when the tunnel wakes — the Cove itself is usually fine.

  1. In the Tailscale app, turn on VPN On Demand for this network (wording varies by version; look under settings for this account).
  2. Allow Tailscale as a VPN in iOS Settings. Avoid Low Power Mode while you need the Cove.
  3. If Mission Control will not load: open Tailscale first. If it is not Connected, fix that before refreshing Safari.
  4. Android: usually stays connected; if it drops, open Tailscale and turn off battery optimizations for the app.
Name this device (#MESH-NAME)

Junk names like localhost-… or invalid-… make family device lists hard to read. Before or right after joining:

  1. Give the device a clear name in system settings (e.g. jade-iphone or beatrice-imac) when the OS allows it — Tailscale often picks that up on join.
  2. On a computer already on the mesh you can rename later: tailscale set --hostname your-device-name

Join codes are single-use and expire in about an hour. Generate a new QR from a signed-in Cove session if this one is stale. After join, give chat a minute if names are still catching up (relay before direct is normal).
