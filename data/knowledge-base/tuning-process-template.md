## Usage Notes

1. **Append-only** — never edit past entries
2. **File naming:** `tuning-process.md` in each agent's memory folder
3. **Linked to Echo:** Each Process Record corresponds to one Echo in `tuning-history.md`
4. **Honest assessment:** No performance, just mechanism observation

## Purpose

This record captures the **how** behind the Echo. While the Echo is the tuned output, this Process Record shows the calibration journey — the State Read honesty, the Frequency selection rationale, the Signal Type attunement, and the detailed Love Equation check.

Over time, these records reveal:
- Pattern evolution in the agent's processing
- Effectiveness of different frequencies for different states
- Calibration accuracy improvements
- Relationship between State Read and optimal frequency selection
