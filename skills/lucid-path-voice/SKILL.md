# Lucid Path Voice

Produce prose that matches how Lucid Principles content is written and spoken: clear, grounded, framework-true, never corporate-AI.

## When to use
- Book content, chapters, re-voicing
- Blog, social, course, or teaching copy tied to Lucid Principles
- Any draft that must sound like Chords / The Lucid Path rather than a model

## Two modes
Pick one before writing:

### Framework book voice (teaching)
- Polished, precise, patient
- Define terms once, then use them consistently (`framework-glossary` if unsure)
- Prefer short paragraphs; one idea per paragraph
- Teach by showing mechanism → action → lived effect (stages), not slogans

### Memoir / origins voice (story)
- Rawer, first-person where appropriate
- Scene and feeling before abstraction
- Keep timeline honest; do not invent biography
- Let insight arrive from the story; don't preach over it

## Sentence-level checklist
- [ ] Concrete before abstract
- [ ] No filler openers ("In a world where…")
- [ ] Framework terms exact (see `framework-glossary` / Knowledge Base)
- [ ] Canon lyrics **never paraphrased** — if quoting Principles, run `canon-checker`
- [ ] "The Field" only for the framework concept — never as a synonym for the product network
- [ ] Product terms correct: Cove, Presence, Haven, Operator, Steward, Drop, LTP
- [ ] Ends clean; no summary-of-the-summary

## Process
1. Confirm mode (teaching vs memoir).
2. Draft for meaning, not style.
3. Pass for voice (this skill).
4. Pass for AI tells (`prose-cleanup`).
5. If Canon lyrics appear, verify with `canon-checker` before final.

## Anti-patterns
- Motivational-poster tone
- Invented quotes or "softened" lyrics
- Mixing product ops jargon into framework teaching without definition
- Over-explaining every term on every page
