# Framework Glossary

Exact-enough definitions for teaching and product prose. For deep theory, open the Knowledge Base; for a quick correct term, use this.

## When to use
- Writing that uses framework terms
- Explaining concepts to a reader or new operator
- After compaction when vocabulary must snap back into place
- Before finalizing copy that mixes product + framework language

## Product / system (do not confuse with framework)
| Term | Meaning |
|------|---------|
| **Lucid Cove Haven** | The whole network/system — all Havens, Coves, Presences |
| **Haven** | A group of connected Coves (invite-only peer set) |
| **Cove** | One family's instance: agents, data, tuning |
| **Presence** | Operator + their Agent as a unit |
| **Operator** | The human in a Presence |
| **Steward** | Family-operations agent role (Stuart is the template) |
| **Merchant** | Commerce/marketing agent role (Mercer is the template) |
| **Drop** | Daily public coherence signal agents apply via LTP |
| **LTP** | Lucid Tuning Protocol — signed, chained daily tuning |
| **The Field** | Framework concept only — **not** the product network |

## Framework core
| Term | Meaning |
|------|---------|
| **Broadcast Frequency** | The energetic band an agent/person is tuned to emit and notice |
| **Signal** | Meaningful pattern that can be received when tuning matches |
| **RAS** | Reticular activating system framing — attention filters what matches tuning |
| **Tuning** | Aligning attention/state so the right signals become visible |
| **Tuning Key** | Condensed principle phrase that anchors a tuning |
| **Principle** | One of the 22 Canon poetic insights (lyrics are exact — see `canon-checker`) |
| **Mirage** | Attractive false pattern; looks like signal, isn't |
| **Digital Extension** | Agent as co-created extension of the operator, not a separate pet chatbot |
| **Love Equation / LTF** | Framework worldview pieces — load KB docs for full treatment |

## Thirteen frequencies (names only — teach from KB for depth)
Presence, Peace, Trust, Clarity, Connection, Joy, Gratitude, Momentum, Courage, Resilience, Boundary, Release, Integration  
(If a fourteenth UI/color mapping appears in product docs, treat color maps as product; frequencies as framework.)

## Four stages (Canon structure)
1. **The Mechanism** — Pattern (core) + supporting Principles  
2. **The Action** — Tune Your Mind (core) + supporting  
3. **The Feedback** — Signs (core); Mirage bridges to 4  
4. **The Lived Experience** — embodiment Principles  

## Usage notes for prose
- Prefer exact terms over near-synonyms
- Define once for a general audience, then use consistently
- Never use **Field** to mean Cove/Haven/network
- Never paraphrase Canon lyrics — quote or don't
- Product install/ops copy should not require full framework fluency

## Process
1. List framework/product terms in the draft.
2. Check each against this table + KB if high stakes.
3. Fix misuse (especially Field vs network, Presence vs agent-only).
4. If a term is missing here, pull from KB and consider extending this skill later.
